# 課程系統重構分析報告

## 📋 系統現況分析

### 1. 現有資料結構

#### 1.1 主要資料表
- **EMTrainingInfo**：訓練資訊主表
  - 儲存每個影片片段的基本資訊
  - 包含：Title, Content, URL, URLTime, Page, Seq 等
  - 目前使用 `Page` 來分組片段

- **EMTrainingDetl**：訓練明細表
  - 記錄員工觀看每個訓練影片的進度
  - 包含：EmployeeMainID, EMTrainingInfoMainID, Time, State
  - **已確認：只需要此表即可完成練習狀態統計**

- **EMTraining**：訓練總表（已不需要）
  - 原本用於統計整體訓練狀態
  - **建議：移除相關程式碼，只使用 EMTrainingDetl**

#### 1.2 新增資料結構需求
- **LessionMain**：主要課程表
  - 包含 `ParagraphJson` 欄位
  - `ParagraphJson` 格式應為：
    ```json
    [
      {
        "LessionInfoID": "Guid",
        "Paragraph": "段落描述"
      }
    ]
    ```
  - 用於繫結主要課程與分段影片

### 2. 需要調整的項目

#### 2.1 樣式統一化
- **問題**：目前使用 `TitleStyle` 和 `ContentStyle` 來設定個別樣式
- **解決方案**：移除所有 style 相關欄位，統一使用 CSS 類別
- **影響範圍**：
  - `PlayMultiSegment.cshtml`：移除 `style="@segment.TitleStyle"` 和 `style="@segment.ContentStyle"`
  - `Create.cshtml`：移除 TitleStyle 和 ContentStyle 輸入欄位
  - `LessionController.cs`：移除 TitleStyle 和 ContentStyle 的處理
  - `LessionRepository.cs`：移除 TitleStyle 和 ContentStyle 的 SQL 欄位
  - `EMTrainingInfo` Model：移除 TitleStyle 和 ContentStyle 屬性

#### 2.2 段落資訊整合
- **問題**：目前使用 `Page` 來分組，無法明確繫結主要課程與分段影片
- **解決方案**：使用 `LessionMain.ParagraphJson` 來管理段落資訊
- **需要實作**：
  1. 建立 `LessionMain` Model
  2. 建立 `LessionMainRepository`
  3. 建立 `LessionMainService`
  4. 修改 `Create` 功能，支援選擇或建立主要課程
  5. 修改 `PlayMultiSegment`，從 `ParagraphJson` 讀取片段資訊

#### 2.3 練習狀態統計簡化
- **問題**：目前有 `EMTraining` 和 `EMTrainingDetl` 兩張表，但實際上只需要 `EMTrainingDetl`
- **解決方案**：移除所有 `EMTraining` 相關程式碼
- **影響範圍**：
  - `LessionRepository.cs`：移除 `GetTrainingAsync` 和 `UpsertTrainingAsync` 方法
  - `LessionService.cs`：移除 `EMTraining` 相關方法
  - `LessionMainModel.cs`：移除 `EMTraining` Model（或保留但標記為已棄用）

## 📝 資料庫結構建議

### LessionMain 資料表結構
```sql
CREATE TABLE [Lession].[dbo].[LessionMain] (
    ID UNIQUEIDENTIFIER PRIMARY KEY,
    Seq BIGINT,
    Title NVARCHAR(500) NOT NULL,
    Content NVARCHAR(MAX),
    ParagraphJson NVARCHAR(MAX), -- JSON 格式：[{LessionInfoID, Paragraph}]
    CreateDate DATETIME,
    CreateUser UNIQUEIDENTIFIER,
    CreateDept UNIQUEIDENTIFIER,
    ModifyDate DATETIME,
    ModifyUser UNIQUEIDENTIFIER,
    ModifyDept UNIQUEIDENTIFIER,
    Enabled BIT DEFAULT 1,
    Deleted BIT DEFAULT 0
);
```

### ParagraphJson 格式範例
```json
[
  {
    "LessionInfoID": "123e4567-e89b-12d3-a456-426614174000",
    "Paragraph": "第一章：基礎介紹"
  },
  {
    "LessionInfoID": "223e4567-e89b-12d3-a456-426614174001",
    "Paragraph": "第二章：進階應用"
  }
]
```

## 🎯 重構目標

1. **統一風格**：所有課程標題和內容使用一致的樣式
2. **結構化段落**：使用 JSON 格式管理段落資訊，便於維護和擴展
3. **簡化統計**：只使用 `EMTrainingDetl` 來統計練習狀態
4. **易於維護**：清晰的資料結構和程式碼組織
