## 🔍 關鍵演算法

### 1. 加法題目生成演算法
```
1. 根據位數範圍產生指定數量的隨機數字
2. 計算總和
3. 檢查總和是否符合結果位數限制
4. 如果不符合：
   a. 重試最多100次
   b. 若仍不符合，降低數字範圍
5. 產生垂直排列格式
```

**時間複雜度:** O(n × m)，其中 n 是題數，m 是重試次數  
**空間複雜度:** O(n)

### 2. 減法題目生成演算法
```
1. 產生被減數（第一個數字）
2. 產生指定數量的減數
3. 計算差值
4. 檢查差值是否符合位數限制和正負數設定
5. 如果不符合：
   a. 調整策略（確保不為負數或允許負數）
   b. 重新計算減數
6. 產生垂直排列格式
```

**特殊處理:** 
- 不允許負數時：確保被減數 ≥ 所有減數的總和
- 允許負數時：放寬限制，只檢查結果位數

### 3. 乘法題目生成演算法
```
1. 根據位數範圍產生被乘數和乘數
2. 計算乘積
3. 檢查乘積是否符合位數限制且 ≥ 10（至少2位數）
4. 如果不符合：
   a. 使用平方根估算調整範圍
   b. 重新產生數字
5. 產生垂直排列格式
```

**優化策略:**
- 使用 `√maxResult` 來估算最大可能的因數
- 確保結果至少為2位數（符合乘法特性）

### 4. 除法題目生成演算法

#### 整除模式
```
1. 產生除數（避免為0）
2. 根據位數要求產生商
3. 計算被除數 = 除數 × 商
4. 產生題目格式：「被除數 ÷ 除數 = ?」
```

#### 有餘數模式
```
1. 根據位數範圍產生被除數
2. 產生除數（避免為0）
3. 計算商和餘數
4. 如果餘數為0：調整被除數確保有餘數
5. 產生題目格式：「被除數 ÷ 除數 = ? ... ?」
```

## 📊 資料庫設計

### 題目儲存格式

#### Vocabulary 資料表
| 欄位名稱 | 資料類型 | 說明 | 範例 |
|---------|---------|------|------|
| WordID | Guid | 題目唯一識別碼 | `3fa85f64-...` |
| CategoryType | String | 運算類型 | `Addition` / `Subtraction` / `Multiplication` / `Division` |
| Question | Text | 題目內容（含換行） | `"  123\n+ 234\n-------"` |
| Answer | String | 答案 | `"357"` / `"25 ... 2"` |
| ClassName | String | 課程名稱 | `Addition_3digit_20250103_143052` |
| TestType | String | 測驗類型 | `Math` |
| LessionID | Guid | 課程ID | `2ba85f64-...` |

### 課程命名規範
```
加法：Addition_{位數}digit_{時間戳}
減法：Subtraction_{位數}digit_{時間戳}
乘法：Multiplication_{被乘數}x{乘數}_{時間戳}
除法：Division_{被除數}div{除數}_{類型}_{時間戳}
```

### 去重機制
系統會檢查 `KidTestIndex` 表來判斷：
- 相同學生ID
- 相同課程名稱
- 相同測驗類型

如果已存在，直接載入之前的題目，避免重複產生。

## 🎓 開發者筆記

### 遵循的設計原則
1. **單一職責原則 (SRP)**
   - 每個方法只負責一件事
   - Controller 只處理 HTTP 請求
   - Service 只處理業務邏輯
   - Repository 只處理資料存取

2. **開放封閉原則 (OCP)**
   - 易於擴展新的運算類型
   - 不需修改現有程式碼

3. **依賴反轉原則 (DIP)**
   - Controller 依賴 Service 介面
   - Service 依賴 Repository 介面

4. **DRY 原則**
   - 共用的顯示元件 (`_ArithmeticTest.cshtml`)
   - 重用現有的資料庫架構

### 編碼風格
- ✅ 繁體中文註解
- ✅ Pascal Case 命名（類別、方法）
- ✅ camelCase 命名（變數）
- ✅ 有意義的變數名稱
- ✅ 適當的空白和縮排
- ✅ 每個方法前有 XML 註解

### 相依套件
- **ASP.NET Core MVC** - Web 框架
- **Dapper** - ORM
- **Microsoft.Data.SqlClient** - 資料庫連線
- **Bootstrap 5** - UI 框架（現有）
- **jQuery** - DOM 操作（現有）
