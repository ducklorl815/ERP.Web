## 相容性說明

### Microsoft.Data.SqlClient vs System.Data.SqlClient
雖然我們更換了 SQL Client 套件，但兩者的 API 幾乎完全相同：
- ✅ `SqlConnection` 類別保持不變
- ✅ `SqlCommand` 類別保持不變
- ✅ `SqlDataReader` 類別保持不變
- ✅ 連線字串格式保持不變
- ✅ Dapper 完全支援 Microsoft.Data.SqlClient

**主要變更：**
1. **命名空間：** 從 `System.Data.SqlClient` 改為 `Microsoft.Data.SqlClient`
2. **連線字串：** 需要添加 `TrustServerCertificate=True` 參數（開發環境）

### ⚠️ 已知問題：SSL 憑證驗證

#### 問題描述
更新後可能遇到以下錯誤：
```
Microsoft.Data.SqlClient.SqlException: 
'A connection was successfully established with the server, but then an error occurred 
during the login process. (provider: SSL Provider, error: 0 - 此憑證鏈結是由不受信任的授權單位發出的。)'
```

#### 原因
`Microsoft.Data.SqlClient` 預設啟用更嚴格的 SSL/TLS 安全性設定：
- 預設啟用連線加密
- 預設要求驗證伺服器 SSL 憑證
- 不接受自簽憑證或不受信任的憑證

#### 解決方案

**開發環境（推薦）：**
在連線字串中添加 `TrustServerCertificate=True`：
```
Data Source=SERVER;Initial Catalog=DB;User ID=user;Password=pwd;TrustServerCertificate=True
```

**正式環境（最佳實踐）：**
1. 使用正式的 SSL 憑證（從受信任的 CA 購買）
2. 或在伺服器上正確配置 SSL 憑證
3. 保持 `TrustServerCertificate=False` 以確保最高安全性

**不推薦的方案：**
```
Encrypt=False  // 會停用加密，降低安全性
```

#### 已套用的修正
- ✅ `appsettings.Development.json` - 已添加 `TrustServerCertificate=True`
