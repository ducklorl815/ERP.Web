# 權限控制系統使用說明

## 📋 概述

本系統提供完整的權限檢查機制，讓頁面能夠根據使用者權限自動顯示或隱藏功能按鈕/連結。

**核心元件：**
1. **IPermissionService** - 權限檢查服務介面
2. **PermissionService** - 權限檢查服務實作（含快取機制）
3. **PermissionTagHelper** - 自訂 TagHelper，在 View 中方便地檢查權限

---

## 💡 使用方式

### 方式 1：完整版（推薦用於複雜場景）

適用於需要明確指定 Controller 和 Action 的場景：

```cshtml
<div permission-controller="ControllerSetting" permission-action="TreeView">
    <a asp-action="TreeView" asp-controller="ControllerSetting">
        <span class="float-right" style="cursor:pointer;">
            <i class="fa-solid fa-plus"></i> 新增群組
        </span>
    </a>
</div>
```

**特點：**
- 明確指定 Controller 和 Action
- 適合需要條件判斷的複雜場景
- 可以包裹多個元素

---

### 方式 2：簡化版（推薦用於簡單場景）

適用於單一連結或按鈕：

```cshtml
<a permission-check="ControllerSetting.TreeView" 
   asp-action="TreeView" 
   asp-controller="ControllerSetting">
    <i class="fa-solid fa-plus"></i> 新增群組
</a>
```

**特點：**
- 簡潔易讀
- 格式：`Controller.Action`
- 適合單一元素

---

### 方式 3：包裹多個元素

當多個元素需要相同權限時：

```cshtml
<div permission-check="ControllerSetting.DeleteAccessGroup">
    <button onclick="DeleteAccessGroup('@item.ID')" class="btn btn-danger btn-sm">
        <i class="fa fa-trash"></i> 刪除
    </button>
    <button onclick="EditAccessGroup('@item.ID')" class="btn btn-warning btn-sm">
        <i class="fa fa-edit"></i> 編輯
    </button>
</div>
```

## 🔧 進階選項

### 保留元素但禁用

預設情況下，沒有權限時會完全移除元素。如果您希望保留元素但禁用：

```cshtml
<button permission-check="ControllerSetting.TreeView" 
        permission-remove="false"
        class="btn btn-primary">
    新增群組
</button>
```

**效果：**
- 沒有權限時：元素保留但添加 `disabled` 屬性和 `display:none` 樣式
- 有權限時：正常顯示

## 🎯 Controller Action 命名規範

為了正確檢查權限，請確保：

1. **Controller 名稱**：不包含 "Controller" 後綴
   - ✅ 正確：`ControllerSetting`
   - ❌ 錯誤：`ControllerSettingController`

2. **Action 名稱**：使用實際的 Action 方法名稱
   - ✅ 正確：`TreeView`、`DeleteAccessGroup`
   - ❌ 錯誤：`tree-view`、`delete_access_group`

3. **格式一致性**：
   ```cshtml
   <!-- 這兩者必須一致 -->
   <a permission-check="ControllerSetting.TreeView"
      asp-action="TreeView" 
      asp-controller="ControllerSetting">
   ```

## 🔍 權限檢查邏輯

### 檢查流程

1. **取得使用者資訊**
   - 從 `HttpContext.User.Identity.Name` 取得當前使用者

2. **查詢權限清單**
   - 從資料庫查詢使用者所屬的權限群組
   - 展開樹狀結構取得所有 Controller ID
   - 查詢對應的 Controller 和 Action

3. **快取機制**
   - 權限清單會快取 30 分鐘
   - 減少資料庫查詢，提升效能
   - 滑動過期時間：10 分鐘無存取則清除

4. **權限比對**
   - 格式：`Controller.Action`（不區分大小寫）
   - 例如：`controllersetting.treeview`

## 🛠️ 在 Controller 中使用權限服務

除了在 View 中使用 TagHelper，您也可以在 Controller 中直接使用權限服務：

```csharp
using ERP.Web.Utility.Services;
using Microsoft.AspNetCore.Mvc;

public class ControllerSettingController : Controller
{
    private readonly IPermissionService _permissionService;

    public ControllerSettingController(IPermissionService permissionService)
    {
        _permissionService = permissionService;
    }

    public async Task<IActionResult> AccessGroupSearchList()
    {
        // 檢查使用者是否有「新增群組」的權限
        var canAddGroup = await _permissionService.HasPermissionAsync(
            User.Identity.Name, 
            "ControllerSetting", 
            "TreeView"
        );

        ViewBag.CanAddGroup = canAddGroup;

        return View();
    }

    [HttpPost]
    public async Task<IActionResult> DeleteAccessGroup(Guid id)
    {
        // 檢查權限
        var hasPermission = await _permissionService.HasPermissionAsync(
            User.Identity.Name, 
            "ControllerSetting", 
            "DeleteAccessGroup"
        );

        if (!hasPermission)
        {
            return Forbid(); // 403 Forbidden
        }

        // 執行刪除邏輯...
        return Ok();
    }
}
```

### 取得使用者所有權限

```csharp
public async Task<IActionResult> GetUserPermissions()
{
    var permissions = await _permissionService.GetUserPermissionsAsync(User.Identity.Name);
    
    // permissions 是 HashSet<string>
    // 包含格式如：["controllersetting.treeview", "controllersetting.delete", ...]
    
    return Json(permissions);
}
```

## 🎨 最佳實踐

### 1. 選擇合適的使用方式

**使用 TagHelper（View）：**
- ✅ 單純的顯示/隱藏邏輯
- ✅ 不影響業務邏輯
- ✅ 減少 Controller 程式碼

**使用 Service（Controller）：**
- ✅ 需要在業務邏輯中判斷權限
- ✅ 需要回傳不同的結果
- ✅ 需要記錄權限檢查日誌

### 2. 權限細粒度

建議為每個重要操作設定獨立的權限：
- ✅ `ControllerSetting.Create` - 新增
- ✅ `ControllerSetting.Edit` - 編輯
- ✅ `ControllerSetting.Delete` - 刪除
- ✅ `ControllerSetting.View` - 查看
- ✅ `ControllerSetting.Export` - 匯出

### 3. 快取管理

權限變更後需要清除快取：

```csharp
public class PermissionManagementController : Controller
{
    private readonly IPermissionService _permissionService;

    [HttpPost]
    public async Task<IActionResult> UpdateUserPermission(string userName)
    {
        // 更新權限邏輯...
        
        // 清除該使用者的權限快取
        if (_permissionService is PermissionService permissionService)
        {
            permissionService.ClearUserPermissionsCache(userName);
        }
        
        return Ok();
    }
}
```

### 4. 效能考量

- ✅ 權限服務使用 Singleton 生命週期
- ✅ 內建記憶體快取（30分鐘）
- ✅ 滑動過期機制（10分鐘）
- ✅ 一次查詢，多次使用

## 🐛 故障排除

### 問題 1：TagHelper 不生效

**檢查清單：**
1. ✅ 確認 `_ViewImports.cshtml` 中有 `@addTagHelper *, ERP.Web.Utility`
2. ✅ 確認 `Program.cs` 中已註冊服務
3. ✅ 重新建置專案（`dotnet build`）
4. ✅ 清除瀏覽器快取

### 問題 2：所有元素都被隱藏

**可能原因：**
1. 使用者未登入（`User.Identity.Name` 為 null）
2. 使用者沒有任何權限
3. Controller/Action 名稱不符合

**解決方式：**
```cshtml
<!-- 調試用：顯示當前使用者 -->
<p>當前使用者：@User.Identity.Name</p>
<p>是否已認證：@User.Identity.IsAuthenticated</p>
```

### 問題 3：權限更新後沒有立即生效

**原因：** 快取機制

**解決方式：**
1. 等待快取過期（最多 30 分鐘）
2. 或在權限更新後清除快取（見上方「快取管理」）
3. 或重新啟動應用程式

### 問題 4：格式錯誤

**常見錯誤：**
```cshtml
<!-- ❌ 錯誤：包含 Controller 後綴 -->
<div permission-check="ControllerSettingController.TreeView">

<!-- ❌ 錯誤：使用分隔符號 -->
<div permission-check="ControllerSetting/TreeView">

<!-- ❌ 錯誤：使用中文 -->
<div permission-check="控制器設定.樹狀檢視">

<!-- ✅ 正確 -->
<div permission-check="ControllerSetting.TreeView">
```

## 🔐 安全性考量

1. **前端檢查不能替代後端驗證**
   - TagHelper 只控制顯示
   - Controller 中仍需驗證權限

2. **敏感操作必須雙重驗證**
   ```cshtml
   <!-- View 層 -->
   <button permission-check="Admin.DeleteUser">刪除</button>
   ```
   
   ```csharp
   // Controller 層
   [HttpPost]
   public async Task<IActionResult> DeleteUser(Guid id)
   {
       if (!await _permissionService.HasPermissionAsync(User.Identity.Name, "Admin", "DeleteUser"))
           return Forbid();
       
       // 執行刪除...
   }
   ```

3. **權限資料的快取時間**
   - 預設 30 分鐘
   - 根據安全性需求調整
