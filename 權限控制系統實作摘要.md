**位置：** `ERP.Web.Utility/Services/IPermissionService.cs`

- 提供三種權限檢查方法

**主要方法：**
```csharp
// 檢查使用者是否有特定 Controller.Action 的權限
Task<bool> HasPermissionAsync(string userName, string controller, string action);

// 取得使用者的所有權限清單
Task<HashSet<string>> GetUserPermissionsAsync(string userName);

// 根據 ID 檢查權限
Task<bool> HasPermissionByIdAsync(string userName, Guid controllerMainID);
```

#### ✅ PermissionService.cs
**位置：** `ERP.Web.Utility/Services/PermissionService.cs`

- 絕對過期：30 分鐘

**效能優化：**
- 一次查詢，多次使用
- 減少資料庫負載
- 支援清除快取（權限變更時）

#### ✅ PermissionTagHelper.cs
**位置：** `ERP.Web.Utility/TagHelpers/PermissionTagHelper.cs`

**功能：**
- 自訂 TagHelper，在 View 中方便檢查權限
- 支援兩種使用方式（完整版和簡化版）
- 可選擇移除元素或僅禁用

**使用方式：**

**方法 1 - 完整版：**
```cshtml
<div permission-controller="ControllerSetting" permission-action="TreeView">
    <a asp-action="TreeView">新增群組</a>
</div>
```

**方法 2 - 簡化版（推薦）：**
```cshtml
<a permission-check="ControllerSetting.TreeView" asp-action="TreeView">
    新增群組
</a>
```

**選項：**
```cshtml
<!-- 沒有權限時僅禁用（不完全移除） -->
<button permission-check="ControllerSetting.Create" permission-remove="false">
    新增
</button>
```

## 🔄 工作流程

### 使用者請求頁面
```
1. 使用者開啟頁面（例如：AccessGroupSearchList）
   ↓
2. LeftSidebarViewComponent 載入（顯示左側選單）
   - 查詢使用者權限
   - 建立選單樹狀結構
   - 標記當前頁面
   ↓
3. 頁面主體載入
   ↓
4. PermissionTagHelper 執行
   - 對每個帶 permission-check 的元素
   - 呼叫 PermissionService.HasPermissionAsync()
   - 判斷是否顯示/隱藏元素
   ↓
5. 頁面渲染完成
   - 有權限的功能顯示
   - 沒有權限的功能隱藏
```

### 權限檢查流程
```
1. TagHelper 觸發權限檢查
   ↓
2. PermissionService.HasPermissionAsync()
   ↓
3. 檢查快取
   - 有快取 → 直接返回結果
   - 無快取 → 執行步驟 4
   ↓
4. 從資料庫查詢
   - ControllerUtilityRepo.GetBoundAccessGroupData() - 取得群組節點
   - ControllerUtilityRepo.GetMenuDataAsync() - 取得選單資料
   ↓
5. 建立權限清單（Controller.Action格式）
   ↓
6. 快取結果（30分鐘）
   ↓
7. 返回權限檢查結果
   ↓
8. TagHelper 根據結果顯示/隱藏元素
```

## 🔧 維護指南

### 新增權限時
1. 在資料庫的 `ControllerMain` 表中新增記錄
2. 將新功能加入適當的權限群組
3. 在 View 中使用 `permission-check` 標籤
4. （可選）在 Controller 中加入權限驗證
5. 更新本文件的權限清單

### 權限變更後
1. 清除受影響使用者的快取
2. 或等待快取自動過期（最多30分鐘）
3. 或重新啟動應用程式

### 除錯時
1. 檢查 `User.Identity.Name` 是否正確
2. 檢查資料庫中的權限設定
3. 檢查快取是否過期
4. 確認 Controller/Action 名稱正確
5. 查看瀏覽器開發者工具（元素是否被移除）

## 📊 效能指標

### 記憶體快取
- **快取類型：** IMemoryCache
- **生命週期：** Singleton
- **過期策略：** 絕對過期 30 分鐘 + 滑動過期 10 分鐘
- **儲存格式：** `HashSet<string>`（Controller.Action 格式）

### 預期效能
- **首次查詢：** ~100-300ms（資料庫查詢 + 樹狀結構處理）
- **快取命中：** <1ms（記憶體存取）
- **記憶體使用：** 每使用者約 1-5KB（取決於權限數量）

## 🔐 安全性考量

### 已實作
✅ 前端權限檢查（TagHelper）
✅ 記憶體快取（不洩漏敏感資料）
✅ 使用者隔離（每個使用者獨立快取）

### 建議加強

⚠️ 操作日誌記錄

- 快取時間不宜過長（安全性 vs 效能）
