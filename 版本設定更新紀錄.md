# 套件安全性更新

## 2025年10月28日 - 套件安全性更新

### 更新概述
解決 ERP.Web.Utility 專案中的安全性弱點警告，並將過時的套件更新至最新穩定版本。

### 已解決的安全性問題

#### 1. Newtonsoft.Json 12.0.3 → 13.0.3
**嚴重性：高 (High)**
- **問題描述：** 舊版本存在已知的高嚴重性弱點 (GHSA-5crp-9r3c-p9vr)
- **解決方案：** 更新至 Newtonsoft.Json 13.0.3
- **影響範圍：** ERP.Web.Utility 專案
- **相容性：** 完全向後相容，API 無重大變更

#### 2. System.Data.SqlClient 4.8.1 → Microsoft.Data.SqlClient 5.1.6
**嚴重性：高 + 中度 (High + Medium)**
- **問題描述：** System.Data.SqlClient 4.8.1 存在兩個已知安全性弱點
  - GHSA-8g2p-5pqh-5jmc (中度嚴重性)
  - GHSA-98g6-xh36-x2p7 (高嚴重性)
- **解決方案：** 改用 Microsoft 官方推薦的新版 SQL Server 資料提供者
- **影響範圍：** 所有使用資料庫連線的專案
- **程式碼變更：** 需要將 `using System.Data.SqlClient;` 改為 `using Microsoft.Data.SqlClient;`

#### 3. 其他套件更新
- **Dapper 2.0.35 → 2.1.35**
  - 更新原因：版本過舊，建議更新至最新穩定版
  - 變更內容：效能改進和錯誤修復
  - 相容性：完全向後相容

- **Microsoft.AspNetCore.Mvc.Razor.RuntimeCompilation 6.0.25 → 6.0.36**
  - 更新原因：安全性修補和錯誤修復
  - 變更內容：.NET 6.0 LTS 最新穩定版本
  - 相容性：完全相容

### Microsoft.Data.SqlClient 遷移指南

#### SSL 憑證信任錯誤
**錯誤訊息：**
```
Microsoft.Data.SqlClient.SqlException: 
'A connection was successfully established with the server, but then an error occurred 
during the login process. (provider: SSL Provider, error: 0 - 此憑證鏈結是由不受信任的授權單位發出的。)'
```

#### 快速解決方案

**步驟 1：** 找到連線字串設定檔（`appsettings.json` 或 `appsettings.Development.json`）

**步驟 2：** 在連線字串末尾添加 `;TrustServerCertificate=True`

**修改前：**
```json
{
  "ConnectionStrings": {
    "erp": "Data Source=SERVER;Initial Catalog=DB;User ID=user;Password=pwd"
  }
}
```

**修改後：**
```json
{
  "ConnectionStrings": {
    "erp": "Data Source=SERVER;Initial Catalog=DB;User ID=user;Password=pwd;TrustServerCertificate=True"
  }
}
```

#### 為什麼需要這個設定？

| 項目 | System.Data.SqlClient (舊) | Microsoft.Data.SqlClient (新) |
|-----|---------------------------|------------------------------|
| SSL/TLS 加密 | 預設關閉 | **預設開啟** |
| 憑證驗證 | 不嚴格 | **嚴格驗證** |
| 自簽憑證 | 接受 | **拒絕（除非明確信任）** |

### 環境設定建議

#### 開發環境 (Development)
```json
{
  "ConnectionStrings": {
    "Default": "Server=localhost;Database=DevDB;User Id=dev;Password=dev123;TrustServerCertificate=True"
  }
}
```
- ✅ 使用 `TrustServerCertificate=True`

#### 正式環境 (Production)
```json
{
  "ConnectionStrings": {
    "Default": "Server=prod-server.company.com;Database=ProdDB;User Id=appuser;Password=SecurePass123!"
  }
}
```
- ✅ 必須使用正式 SSL 憑證
- ❌ 不要使用 `TrustServerCertificate=True`

# 資料庫相關更新

## SQL Client 套件更新（2025年10月28日）

### 必須使用的套件
```xml
<PackageReference Include="Microsoft.Data.SqlClient" Version="5.1.6" />
```

### 禁止使用的套件
```xml
<!-- 已棄用且有安全性弱點 -->
<PackageReference Include="System.Data.SqlClient" Version="4.8.1" />
```

### 遷移步驟
1. 更新 .csproj 檔案中的套件參考
2. 將所有 `using System.Data.SqlClient;` 改為 `using Microsoft.Data.SqlClient;`
3. 在連線字串中添加 `TrustServerCertificate=True`（開發環境）
4. API 使用方式保持不變（SqlConnection、SqlCommand、SqlDataReader 等）

### 連線字串設定
```json
// 開發環境：必須添加 TrustServerCertificate=True
"erp": "Data Source=SERVER;Initial Catalog=DB;User ID=user;Password=pwd;TrustServerCertificate=True"

// 正式環境：使用正式 SSL 憑證，保持 TrustServerCertificate=False
"erp": "Data Source=prod-server.company.com;Initial Catalog=ProdDB;User ID=appuser;Password=SecurePass!"
```
