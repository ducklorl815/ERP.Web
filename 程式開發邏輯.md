# 登入系統

## 登入系統架構

### 三層式架構
```
Controller 層 (AccountController)
    ↓
Service 層 (AccountLoginService)
    ↓
Repository 層 (AccountLoginRepo) ← SQL 實作
```

### 已建立的檔案
1. **Repository 層**
   - `ERP.Web.Models\Respository\Account\AccountLoginRepo.cs`
   - 負責所有資料庫操作

2. **Service 層**
   - `ERP.Web.Service\Service\Account\AccountLoginService.cs`
   - 負責業務邏輯處理

3. **Controller 層**
   - `ERP.Web\Controllers\Account\AccountController.Login.cs`
   - 負責 HTTP 請求處理

## 登入流程

### 手動登入流程
```
使用者輸入帳密
    ↓
AccountController.Login (POST)
    ↓
AccountLoginService.LoginAsync()
    ├─ ValidateUserAsync() → SQL 驗證帳密
    ├─ GetUserDataAsync() → SQL 查詢使用者資料
    └─ UpdateLastLoginTimeAsync() → SQL 更新時間
    ↓
InitializeUserSessionAsync()
    ├─ 設定 Session (UserAccount, UserName, UserEmpID, LoginTime)
    └─ PermissionService.GetUserPermissionsAsync() → 權限寫入 Cookie
    ↓
HandleAutoLoginCookie()
    └─ 勾選「自動登入」→ 寫入 AutoLogin_Account Cookie
    ↓
導向 Home/Index
```

### 自動登入流程
```
使用者進入 /Account/Login
    ↓
檢查 AutoLogin Cookie
    ├─ 無 → 顯示登入表單
    └─ 有 ↓
PerformAutoLoginAsync()
    ↓
AccountLoginService.GetUserDataByAccountAsync()
    └─ AccountLoginRepo.GetUserDataAsync() → SQL 查詢
    ↓
InitializeUserSessionAsync()
    ├─ 設定 Session
    └─ 初始化權限
    ↓
導向 Home/Index
```

## Session 與 Cookie 資料

### Session 資料（存在伺服器）
| Key | 型別 | 說明 | 設定時機 |
|-----|------|------|----------|
| `UserAccount` | string | 使用者帳號 | 登入成功後 |
| `UserName` | string | 員工姓名 | 登入成功後 |
| `UserEmpID` | string | 員工ID | 登入成功後 |
| `LoginTime` | string | 登入時間 | 登入成功後 |
| `PermissionCount` | int | 權限數量 | 權限初始化後 |

### Cookie 資料（存在用戶端）
| Cookie Name | 用途 | 有效期 | 設定時機 |
|------------|------|--------|----------|
| `AutoLogin_Account` | 記錄帳號（自動登入） | 30 天 | 勾選「自動登入」 |
| `UserPermissions_{account}` | 權限清單 | 30 分鐘 | 登入成功後（自動） |

## Repository 層需要實作的 SQL

### 1. ValidateUserAsync - 驗證帳密
```sql
SELECT COUNT(*) 
FROM Users 
WHERE Account = @Account 
  AND Password = HASHBYTES('SHA2_256', @Password + Salt)
  AND IsActive = 1
```

### 2. GetUserDataAsync - 取得使用者資料
```sql
SELECT 
    Account,
    EmpID,
    EmpName,
    Department,
    Email,
    IsActive,
    LastLoginTime
FROM Users
WHERE Account = @Account
  AND IsActive = 1
```

### 3. UpdateLastLoginTimeAsync - 更新登入時間
```sql
UPDATE Users 
SET LastLoginTime = GETDATE(),
    LoginCount = LoginCount + 1
WHERE Account = @Account
```

### 4. LogFailedLoginAsync - 記錄登入失敗
```sql
INSERT INTO LoginFailedLog (Account, Reason, AttemptTime, IPAddress)
VALUES (@Account, @Reason, GETDATE(), @IPAddress)
```

## 中文編碼問題解決

### 問題
錯誤訊息顯示為 HTML 實體編碼：
```
&#x5E33;&#x865F;&#x6216;&#x5BC6;&#x78BC;&#x932F;&#x8AA4;
```

### 解決方案
使用 JSON 序列化 + .text() 方法：
```javascript
// JavaScript: 使用 JSON 序列化安全傳遞中文
var errorMsg = @Html.Raw(System.Text.Json.JsonSerializer.Serialize(TempData["ErrorMessage"]?.ToString() ?? ""));
if (errorMsg && errorMsg.trim() !== '') {
    $('#errorMessage').text(errorMsg).show();
}
```

# 權限控制系統

## 核心元件
1. **IPermissionService** - 權限檢查服務介面
2. **PermissionService** - 權限檢查服務實作（含快取機制）
3. **PermissionTagHelper** - 自訂 TagHelper，在 View 中檢查權限

## 服務註冊
```csharp
// Program.cs
builder.Services.AddMemoryCache();
builder.Services.AddHttpContextAccessor();
builder.Services.AddSingleton<IPermissionService, PermissionService>();
```

## 使用方式

### 方式 1：簡化版（推薦）
```cshtml
<a permission-check="ControllerSetting.TreeView" 
   asp-action="TreeView" 
   asp-controller="ControllerSetting">
    <i class="fa-solid fa-plus"></i> 新增群組
</a>
```

### 方式 2：完整版
```cshtml
<div permission-controller="ControllerSetting" permission-action="TreeView">
    <a asp-action="TreeView" asp-controller="ControllerSetting">
        <span class="float-right" style="cursor:pointer;">
            <i class="fa-solid fa-plus"></i> 新增群組
        </span>
    </a>
</div>
```

### 方式 3：在 Controller 中使用
```csharp
public class ControllerSettingController : Controller
{
    private readonly IPermissionService _permissionService;

    public async Task<IActionResult> DeleteAccessGroup(Guid id)
    {
        // 檢查權限
        var hasPermission = await _permissionService.HasPermissionAsync(
            User.Identity.Name, 
            "ControllerSetting", 
            "DeleteAccessGroup"
        );

        if (!hasPermission)
        {
            return Forbid(); // 403 Forbidden
        }

        // 執行刪除邏輯...
        return Ok();
    }
}
```

## 權限檢查流程
```
1. TagHelper 觸發權限檢查
   ↓
2. PermissionService.HasPermissionAsync()
   ↓
3. 檢查快取
   - 有快取 → 直接返回結果
   - 無快取 → 執行步驟 4
   ↓
4. 從資料庫查詢
   - GetBoundAccessGroupData() - 取得群組節點
   - GetMenuDataAsync() - 取得選單資料
   ↓
5. 建立權限清單（Controller.Action格式）
   ↓
6. 快取結果（30分鐘）
   ↓
7. 返回權限檢查結果
```

## 快取策略
- 快取 Key：`UserPermissions_{userName}`
- 絕對過期：30 分鐘
- 滑動過期：10 分鐘（無存取則清除）

## Controller 架構

### ExamEnglishController
- 固定 TestType = "English"
- 負責英文考試相關功能

**主要 Action：**

### ExamMathController
- 固定 TestType = "Math"
- 負責數學考試相關功能

**主要 Action：**
- `NewTest` - 新測驗頁面
- `ReTest` - 複習測驗頁面
- `Test` - 考試頁面
- `ReExam` - 複習考試頁面

- `UpdateNewTestWord` - 更新題目內容
- `GetTestDates` - 取得考試日期

## TestType 篩選機制

### 雙重篩選策略

#### 第一層：Repository 層（SQL 查詢）
在 SQL 查詢中加入 TestType 篩選：
```sql
-- 修改的方法
GetNewTestCountAsync
GetNewTestListAsync
GetReTestCountAsync
GetReTestSearchListAsync

-- SQL 篩選條件
AND les.TestType = @TestType
```

#### 第二層：Service 層（記憶體篩選）
篩選課程清單（下拉選單）：
```csharp
// PublicTaskAsync 方法
if (!string.IsNullOrEmpty(param.TestType))
{
    ExamList = ExamList.Where(x => x.TestType.Equals(param.TestType, StringComparison.OrdinalIgnoreCase)).ToList();
}
```

## 資料流程

### ExamEnglishController 流程
```
1. Controller 設定 TestType = "English"
   ↓
2. Service 將 TestType 傳入 ExamKeyword
   ↓
3. Repository SQL 查詢：WHERE les.TestType = 'English'
   ↓
4. Service 篩選課程清單：只保留 English 課程
   ↓
5. 返回給 View（全部都是英文相關）
```

# 系統配置

## Program.cs 配置

### Session 支援
```csharp
// Session 支援（登入功能需要）
builder.Services.AddDistributedMemoryCache();
builder.Services.AddSession(options =>
{
    options.IdleTimeout = TimeSpan.FromMinutes(30); // Session 閒置 30 分鐘後過期
    options.Cookie.HttpOnly = true;
    options.Cookie.IsEssential = true;
    options.Cookie.Name = ".ERP.Session";
});
```

### 服務註冊
```csharp
// 帳號登入服務（Scoped - 每個 HTTP 請求一個實例）
builder.Services.AddScoped<AccountLoginService>();

// 權限服務（Singleton - 全應用程式共用）
builder.Services.AddSingleton<IPermissionService, PermissionService>();
```

### 中介軟體順序
```csharp
app.UseRouting();
app.UseSession();  // 必須在 UseRouting 之後
app.UseAuthorization();
```

## 服務生命週期

| 服務 | 生命週期 | 說明 |
|------|----------|------|
| **IPermissionService** | Singleton | 權限服務，全應用程式共用一個實例 |
| **AccountLoginService** | Scoped | 登入服務，每個請求一個實例 |
| **ControllerSettingService** | Singleton | 控制器設定服務 |
| **ExamService** | Singleton | 考試服務 |

# 資料庫存取

## ORM 選擇
- **主要使用：** Dapper 作為 ORM
- **避免使用：** Entity Framework（除非專案已有現有的 EF 實作）

## SQL Client 套件規範

### 必須使用
```xml
<PackageReference Include="Microsoft.Data.SqlClient" Version="5.1.6" />
```

### 禁止使用
```xml
<!-- 已棄用且有安全性弱點 -->
<PackageReference Include="System.Data.SqlClient" Version="4.8.1" />
```

### 命名空間
```csharp
// ✅ 正確
using Microsoft.Data.SqlClient;

// ❌ 錯誤
using System.Data.SqlClient;
```

## 連線字串設定

### 開發環境
```json
{
  "ConnectionStrings": {
    "erp": "Data Source=SERVER;Initial Catalog=DB;User ID=user;Password=pwd;TrustServerCertificate=True"
  }
}
```

**注意：** 必須添加 `TrustServerCertificate=True`

### 正式環境
```json
{
  "ConnectionStrings": {
    "erp": "Data Source=prod-server.company.com;Initial Catalog=ProdDB;User ID=appuser;Password=SecurePass!"
  }
}
```

**注意：** 使用正式 SSL 憑證，保持 `TrustServerCertificate=False`

## SQL 語法規範
- SQL 語法需要使用 **SQL Server 2016** 版本能夠執行的語法
- 避免使用 SQL Server 2016 不支援的新功能
- 使用標準的 T-SQL 語法

## Transaction 交易管理
- 避免分散式交易
- 保持相同的連線字串
- 使用 `using` 來保證資源釋放
- 優先使用 SqlTransaction 而非 TransactionScope（單一資料庫）

## Null 處理規範
- 對可能為 null 的物件進行預防性檢查
- 優先使用空條件運算子 (?.) 和空合併運算子 (??)
- 對於集合類型，使用 null 檢查後再進行操作
